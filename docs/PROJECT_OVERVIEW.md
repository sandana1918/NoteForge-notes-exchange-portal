# Project Documentation

## User Flow

1. Visitor opens the public landing page.
2. Visitor signs up or logs in.
3. Student enters the protected library.
4. Student searches and filters notes.
5. Student previews or downloads files.
6. Student uploads new academic material.
7. Student manages uploads from dashboard.

## Functional Areas

- Authentication
- Notes browsing
- Search and filtering
- File uploads
- File preview and download
- Voting
- Dashboard
- Seeded demo data

## File Preview Behavior

- PDF: inline preview.
- Images: inline preview.
- TXT: inline preview.
- PPTX: extracted slide text preview.
- PPT/DOC/DOCX: file card with download action.
